# 指示
あなたは、優れたUX/UIデザイナーであり、SEOとGoogle AdSense審査に精通したフロントエンドエンジニアです。
現在開発中のコントラクトブリッジ分析Webアプリケーション（SPA）において、AdSense審査を通過するための改修を行ってください。

# 背景と課題
- 現在のサイトはツールとして非常に完成度が高く使いやすいですが、「テキストコンテンツが不足している（独自性がない）」という理由でAdSense審査に落ちてしまいました。
- 審査通過のために、ツールの説明文や必須ページを追加する必要があります。

# 厳守事項（制約）
- **現在のユーザー体験（UX）を絶対に損なわないこと。**
- ファーストビュー（スクロールせずに見える範囲）にあるツールのUI、操作性、レイアウトは現状を維持してください。
- 広告審査用のテキストは、ツール操作の邪魔にならないよう、ツール領域の下部（スクロールした先）に配置するか、別ページに分離してください。

# 実装要求事項

## 1. ツールページ下部へのテキストコンテンツ追加
既存の各ツール（Double Dummy Solver, Single Dummy Solver, Opening Lead Analyzer）のコンポーネントの下部に、以下の構成で解説セクション（テキストエリア）を追加してください。デザインは既存のUIに馴染む、クリーンでシンプルなもの（必要であればアコーディオン形式なども可）にしてください。
- **このツールについて（Overview）:** ツールの目的と強み
- **使い方（How to use）:** 初心者にもわかる具体的な操作手順
- **用語解説（Glossary）:** 該当ツールに関連するブリッジ用語（HCP、Shape、NTなど）の簡単な説明
※ 上記のテキストの初稿（ダミーではなく実際のコントラクトブリッジの知識に基づいた文章）もあなたが生成して組み込んでください。

## 2. AdSense審査必須ページの新規作成
以下の3つの静的ページ（コンポーネント）を新規作成し、グローバルフッターにリンクを設置してください。
- **Privacy Policy（プライバシーポリシー）:** AdSenseやアクセス解析ツールを利用している旨を含む標準的な内容。
- **About Us（運営者情報）:** 「モダンで使いやすいブリッジツールを提供する」というサイトの目的を記載。
- **Contact（お問い合わせ）:** ユーザーが連絡できるフォームの記載欄。

## 3. コラム・解説記事用（ブログ機能）の枠組み作成
審査において「読み物」としての価値を高めるため、`/articles` または `/blog` のようなルーティングを新設してください。
- 記事の一覧ページと、記事詳細ページのコンポーネントの雛形を作成してください。
- サンプルとして、「Double Dummy Solverを活用したハンド分析の実例」というテーマの記事データを1件、ハードコード（またはMarkdown等）で実装して表示させてください。

## 4. ルーティングとSEOメタデータの設定
- 現在のツールがすべて同一URLで動作している場合は、各ツールに独立したURL（例: `/double-dummy`, `/single-dummy`）を割り当ててください。
- 各ページ（ツール、必須ページ、記事）ごとに、適切な `<title>` と `<meta name="description">` が動的に切り替わるように実装してください（react-helmet等を使用）。
