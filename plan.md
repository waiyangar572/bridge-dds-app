あなたはアルゴリズム設計と組み合わせ数学に精通したエンジニアであり、コントラクトブリッジのエキスパートです。
以下の要件とアルゴリズム設計に従い、現在の条件付き確率の計算ツールを修正して。 

# 目的
モンテカルロ法（乱数シミュレーション）を一切使用せず、動的計画法（DP）と組み合わせ数学を用いて、指定された条件に基づくハンドの厳密な確率を求めるツールを構築する。

# 前提条件と制約
- デッキの52枚のカードを個別に扱うと計算爆発を起こすため、「20バケットモデル」を採用する。
  - スート（♠, ♥, ♦, ♣） × ランク（A, K, Q, J, x）の20グループ。
  - `x` は2〜10のスモールカードを指し、HCP（High Card Points）を持たないがシェイプに影響する。
- 判明しているカード（Known Cards）は事前にデッキから引き抜き、各プレイヤーの必要枚数から減算しておく。

# 入力インターフェースのイメージ
- `known_cards`: 誰がどのカードを持っているか（例: N=[♠A, ♥K]）
- `constraints`: 各プレイヤー（N, S, E, W）の HCPレンジ（例: 15-17）および シェイプレンジ（例: ♠5枚以上）
- `targets`: 確率を求めたい未知の条件のリスト（複数指定可能）

# コアエンジンのアルゴリズム設計（実装要件）
以下の要素を持つDP（深さ優先探索 + メモ化）関数 `count_combinations` を実装してください。

1. 状態（State）の定義:
   - `bucket_index`: 現在処理しているバケット（0〜19）
   - `rem_N, rem_S, rem_E`: N, S, Eの残り必要枚数（Wの枚数は自動的に決まるため不要）
   - 各プレイヤーの「現在のHCP」と「各スートの現在の枚数」（制約で指定されている場合のみ追跡し、不要な状態は持たないようにして次元を減らすこと）

2. 状態遷移と組み合わせ計算:
   - 現在のバケットにある残り枚数 `C` 枚を、N, S, E, W に `n, s, e, w` 枚ずつ分配する全パターンをループする（条件: n+s+e+w = C）。
   - 分配パターンの組み合わせ数は、多項係数 `math.comb(C, n) * math.comb(C-n, s) * ...` を用いて計算し、次の状態への遷移結果に乗算する。

3. 枝刈り（Pruning）:
   - 探索の途中で、これ以上カードを配っても指定されたHCPレンジやシェイプレンジに絶対に到達不可能になった場合、その時点で探索を打ち切り `0` を返す。

4. 確率の算出方法:
   - ベースとなる条件（`known_cards` と `constraints`）を満たす全組み合わせ数 $\Omega$ を計算する。
   - `targets` に含まれる各条件について、ベース条件との積集合を満たす組み合わせ数 $A$ をそれぞれ計算する。
   - 戻り値として、各ターゲットの確率 $P = A / \Omega$ を辞書形式などで一覧比較できるように返す。
